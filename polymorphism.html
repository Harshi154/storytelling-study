Polymorphism in Java

Polymorphism in Java is a core object-oriented programming concept that allows a single interface, method, or object to take on multiple forms. Derived from the Greek words poly (many) and morph (forms), it enables developers to perform a single action in different ways based on the context or object invoking it.

Types of Polymorphism in Java

Java supports two primary forms of polymorphism, each resolving method behavior at a different stage of program execution:

1. Compile-Time Polymorphism (Static Binding)

This occurs when the Java compiler resolves which specific method to execute during compilation. It is achieved through Method Overloading.

  • Method Overloading: Defining multiple methods within the same class that share the exact same name but have different parameter signatures (different number, types, or order of arguments).
class Calculator {
    int add(int a, int b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; } // Overloaded
}

2. Runtime Polymorphism (Dynamic Binding)

This occurs when the method call is resolved while the application is executing rather than during compilation. It is achieved through Method Overriding and Upcasting.

  • Method Overriding: When a subclass provides its own specific implementation for a method that is already defined in its parent class (with the exact same signature and return type).
  • Upcasting: Referencing a child class object using a parent class or interface reference variable. The Java Virtual Machine (JVM) dynamically evaluates the actual object type at runtime to call the overridden method.
class Animal {
    void sound() { System.out.println("Animal makes a sound"); }
}

class Dog extends Animal {
    @Override
    void sound() { System.out.println("Bark"); } // Overridden
}

// Usage Context:
Animal myDog = new Dog(); // Upcasting
myDog.sound();            // Outputs "Bark" at runtime dynamically

Core Properties & Characteristics

  • Object Substitution (IS-A Relationship): A parent reference variable can seamlessly hold a reference to any of its child objects. For example, Shape s = new Circle(); is entirely valid.
  • Reference Type Decides Visibility: The type of the reference variable determines which methods can be called on the object. The compiler will throw an error if you try to invoke a subclass-exclusive method using a superclass reference.
  • Execution Decided by Object Instance: While the reference type checks method availability at compile time, the actual instantiated object type determines which version of an overridden method executes at runtime.
  • Coercion (Implicit Type Conversion): Java features internal type casting properties to automatically promote narrower data types to broader data types (e.g., passing an int to a method accepting a double) to avoid type matching errors.
  • Internal Operator Overloading: While Java prevents developers from writing custom operator overloads, the language itself utilizes internal operator polymorphism. A prime example is the + operator, which acts as numeric addition for numbers and string concatenation for strings.

Core Benefits

  • Code Reusability: Write highly generic code using parent types or interfaces that automatically adapt to handle any new child type.
  • Extensibility: Introduce new subclasses to an existing framework without needing to refactor or modify the underlying master code.
  • Loose Coupling: Minimizes interdependencies between classes, allowing systems to remain modular and far easier to maintain.
