## Visibility in Java (Access Modifiers)

Visibility controls the scope of access for variables, methods, constructors, and classes. Java uses **Access Modifiers** to protect sensitive information, achieve encapsulation, and control application architecture constraints.

The Four Main Visibility Levels:

  • private: The strictly narrow scope level. Accessible only inside the containing class structure.
  • default (No keyword specified): Accessible only inside classes within the same package folder. Also termed package-private.
  • protected: Accessible within the same package, plus subclasses residing across completely different package folders.
  • public: The widest scope availability level. Accessible universally from anywhere across the structural program environment.

## Practical Code Implementation

The code blocks below showcase standard data encapsulation techniques utilizing varying visibility limits on fields.

class Account {
    // 1. private visibility level hides sensitive information
    private double balance;
    
    // 2. public constructor allows instance allocation from outside
    public Account(double initialBalance) {
        this.balance = initialBalance;
    }

    // 3. public getter method securely exposes read-only operations
    public double getBalance() {
        return this.balance;
    }

    // 4. protected tracking routine restricted to sub-structures
    protected void logTransaction() {
        System.out.println("Internal visibility log routine invoked.");
    }
}

public class Main {
    public static void main(String[] args) {
        Account acc = new Account(5000.0);
        
        // Direct access fails compilation: acc.balance = 10000;
        
        // Accessing through secure public methods works cleanly
        System.out.println("Current Balance: " + acc.getBalance());
    }
}
