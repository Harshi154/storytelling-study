## What are Java Packages?

A Package in Java is a mechanism used to encapsulate a group of classes, sub-packages, and interfaces. Think of it like a folder directory on your computer that organizes your source files.

Key Purposes & Advantages:

  • Preventing Naming Conflicts: Two classes can have the same exact name (e.g., User) as long as they live in different packages.
  • Access Control: Packages interface closely with access modifiers (like `protected` and default package-private visibility).
  • Modularity and Maintenance: Locating files becomes significantly easier in large production codebases when categorized systematically.
  • Built-in Packages: Java comes equipped with robust internal packages like java.util, java.lang, and java.io.

## Practical Code Implementation

To use custom packages, you define the directory namespace at the absolute top of the source file. Here is how you structure a package definition and import it across separate source files.

// FILE 1: Structured inside directory: ./com/education/math/
package com.education.math;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}
// FILE 2: Structured inside an entirely different directory namespace
package com.education.main;

// Importing the Calculator class from its package
import com.education.math.Calculator;

public class App {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int result = calc.add(15, 25);
        System.out.println("Calculated Value: " + result);
    }
}
