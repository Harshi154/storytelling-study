Object Association Architecture

Association in Java is a structural relationship between two separate classes established through their objects, defining how they interact and use each other's functionality. It is considered a "Uses-A" relationship where both entities possess completely independent lifecycles and no single class acts as the absolute owner of the other.

Core Properties of Association

  • Independent Lifecycles: Associated objects are created and destroyed completely independently. If one object is deleted or garbage collected, the other object continues to exist unaffected.
  • No Ownership: It implies a loose connection or collaboration where neither object owns the other.
  • Directional Nature: It can be Unidirectional (only Class A knows about Class B) or Bidirectional (both classes maintain references to each other).
  • Flexible Cardinality (Multiplicity): The relationship can naturally map to various business rules:
    • One-to-One: e.g., A Person and their Passport.
    • One-to-Many: e.g., A Bank with multiple Employees.
    • Many-to-One: e.g., Multiple City objects belonging to one State.
    • Many-to-Many: e.g., Multiple Student objects interacting with multiple Teacher objects.

The Two Forms of Association

Association is an umbrella term that branches into two specialized forms based on the strength and dependency of the bond:

1. Aggregation (Weak Association)

Represents a "Has-A" relationship. The container object holds a reference to the contained object, but the contained object can survive on its own if the container is destroyed.

Example: A Department has Professors. If the department closes, the professors still exist.

2. Composition (Strong Association)

Represents a "Part-Of" relationship. It is a highly restricted version where the contained object's lifecycle is completely dependent on its parent container.

Example: A House has Rooms. If the house is torn down, the rooms cease to exist.

Feature Association (General) Aggregation (Weak) Composition (Strong)
Relationship Uses-A Has-A Part-of
Lifecycle Entirely independent Independent Strongly dependent
Ownership No ownership Weak / Shared ownership Strict single ownership

Code Implementation Example

The easiest way to implement association in Java is by passing an instance variable of one class into another class. Below is a unidirectional, Many-to-One association between a Driver and a Car.

// Independent Class 1
class Driver {
    private String name;

    public Driver(String name) {
        this.name = name;
    }

    public String getName() {
        return this.name;
    }
}

// Independent Class 2 using the Driver class
class Car {
    private String model;
    private Driver driver; // Association established through an object reference

    public Car(String model, Driver driver) {
        this.model = model;
        this.driver = driver;
    }

    public void displayDetails() {
        // Utilizing functionality/properties of the associated object
        System.out.println("Car model: " + model + " is driven by: " + driver.getName());
    }
}

// Execution block demonstrating independent lifecycles
public class Main {
    public static void main(String[] args) {
        // 1. Create the Driver object independently
        Driver driverObj = new Driver("Alex");

        // 2. Associate the driver object with a Car object
        Car carObj = new Car("Tesla Model 3", driverObj);
        carObj.displayDetails();

        // 3. Destroying the Car object does not destroy the Driver object
        carObj = null; 
        System.out.println("The car is destroyed, but driver " + driverObj.getName() + " still exists.");
    }
}
