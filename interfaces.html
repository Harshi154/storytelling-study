## What is a Java Interface?

An Interface in Java is a blueprint of a class. It is a collection of abstract methods and static constants. It specifies what a class should do, but not how it does it.

Key Characteristics:

  • 100% Abstraction: By default, all methods inside an interface are public and abstract (until Java 8 introduced default and static methods).
  • Multiple Inheritance: A Java class can inherit only one class, but it can implement multiple interfaces.
  • Loose Coupling: Interfaces allow software components to interact without being tightly bound to specific structural implementations.

## Practical Code Implementation

Below is a classic demonstration of multiple interfaces implemented by a single class structure.

// 1. Define the first Interface
interface Swimmer {
    void swim();
}

// 2. Define the second Interface
interface Flyer {
    void fly();
}

// 3. Implementation class combining both blueprints
class Duck implements Swimmer, Flyer {
    public void swim() {
        System.out.println("Duck is paddling smoothly through water.");
    }

    public void fly() {
        System.out.println("Duck is soaring over the lake.");
    }
}

// 4. Executive Main Application
public class Main {
    public static void main(String[] args) {
        Duck mallard = new Duck();
        mallard.swim();
        mallard.fly();
    }
}
