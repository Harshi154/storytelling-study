Understanding Java Objects

In Java, an object is a concrete instance of a class that represents a real-world entity and occupies physical space in your computer's memory. While a class serves as an abstract blueprint or template, an object is the actual, tangible implementation built from that blueprint. For example, if Car is a class, then your specific red sedan is an object.

Every object in Java possesses three core characteristics. These are detailed below, along with how objects behave in memory.

The Three Fundamental Properties of an Object

  • State: This represents the data or attributes stored inside the object. It is defined by instance variables (fields) in the class blueprint. For a Car object, the state might include color = "Red", speed = 60, and fuelLevel = 0.8.
  • Behavior: This refers to the actions or operations that the object can perform. Behaviors are defined via methods (functions) inside the class. For example, a Car object can have behaviors like accelerate(), applyBrakes(), or turnLeft().
  • Identity: This is a unique identifier assigned to each object by the Java Virtual Machine (JVM). Even if two separate objects have the exact same state (e.g., two identical black cars), they have different memory addresses and identities. Internally, this is tracked using a hidden unique ID or hashcode.

Memory Properties: Stack vs. Heap

Java manages object memory dynamically using two distinct areas:

  • The Heap Memory: The actual object data (its state and variables) is stored here. The heap allocates memory at runtime whenever you create an object.
  • The Stack Memory: Holds the reference variable (the name you give to the object in your code). This reference points directly to the location of the real object data sitting in the heap.

How an Object is Created and Used

Objects are typically created using the new keyword, which allocates memory and calls a constructor to initialize the data. You interact with the object's properties using the dot (.) operator.

// 1. Defining the blueprint (Class)
class Car {
    // State / Properties
    String color;
    int speed;

    // Behavior / Method
    void drive() {
        System.out.println("The " + color + " car is moving at " + speed + " mph.");
    }
}

public class Main {
    public static void main(String[] args) {
        // 2. Creating the Object
        // 'myCar' is the reference variable on the stack
        // 'new Car()' allocates space for the actual object on the heap
        Car myCar = new Car();

        // 3. Accessing and changing the State
        myCar.color = "Red";
        myCar.speed = 60;

        // 4. Invoking the Behavior
        myCar.drive(); // Outputs: The Red car is moving at 60 mph.
    }
}

The Ultimate Parent: The Object Class

As a technical rule, Java has a built-in class named java.lang.Object. Every class you write automatically inherits from this master Object class. Because of this inheritance, every single object in Java automatically gains standard built-in properties/methods, such as toString() (to print the object as text), equals() (to compare objects), and hashCode() (to get its unique ID).
